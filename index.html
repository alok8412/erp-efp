Overview

This project explains an ERP (Enterprise Resource Planning) system designed for a manufacturing unit. The goal is to connect different departments—like purchase, inventory, production, sales, and accounts—into one system so data is accurate, fast, and easy to track.

ERP Modules (Main Parts)

  • Purchase: suppliers, purchase orders (PO), material receiving (GRN)
  • Inventory/Warehouse: stock levels, inward/outward, reorder alerts
  • Production: BOM (Bill of Materials), work orders, machine/shift planning
  • Quality: inspections, defect reporting, rework tracking
  • Sales: quotations, sales orders, dispatch, delivery
  • Accounts: invoices, payments, expenses, basic reports
  • HR (optional): attendance, payroll, staff records

Simple Workflow (How It Works)

  1. Customer order comes in → Sales Order created
  2. System checks raw material stock → if low, creates Purchase Request
  3. Materials received into warehouse → stock updated automatically
  4. Production Work Order created using BOM → material issued to shop floor
  5. Quality check happens → pass/rework recorded
  6. Finished goods moved to inventory → dispatch to customer
  7. Invoice generated → payment tracked in accounts

Benefits

  • Reduces manual work and paper records
  • Real-time stock visibility (avoid shortages/overstock)
  • Better production planning and tracking
  • Improves delivery time and customer satisfaction
  • Clear reports for costs, sales, and productivity

Tech / Tools Used

  • Hosting: GitHub Pages
  • Optional (if you extend): Firebase / Supabase / Node.js API
